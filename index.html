SELF-DIRECTED PROTOTYPE · BUILT WITH CLAUDE CODE AS PAIR-PROGRAMMER

Open Banking aggregation + a three-tier GenAI transaction categorizer

After years managing Open Banking from the bank's side, I wanted the TPP developer's side — where the friction is, and where AI actually moves the needle. So I built it. I authored the PRD and made every product, architecture, and integration decision; Claude Code wrote the implementation under my direction. A rare, hands-on test of using AI as a product-development partner.

What it does
  • Integrates five European banks at the sandbox-API level (Nordea, Commerzbank, UniCredit, Deutsche Bank, ING) over Berlin Group / NextGenPSD2 — OAuth2, mTLS, and HTTP Request Signatures (ING the hardest: two key-pairs, two keyId formats).
  • Multi-tenant Flask app with per-user bank connections, multi-currency aggregation via live ECB rates, consent + SCA flows.
  • Ships a GenAI transaction categorizer — a three-tier hybrid router: hand-curated overrides → SQLite cache → a local LLM (Ollama + Qwen 2.5 3B) returning structured JSON {category, confidence, reasoning}.
What shipped
  • Unclassified transactions cut ~80% — from 217 to 42 of 1,303 — measured on an evaluation harness comparing rule-based vs AI categorization.
  • Caught a real LLM bias bug in evaluation (merchant names mis-routed to the wrong category) and fixed it through the override layer — the durable "LLM + overrides" pattern.
  • Splunk-shaped structured JSON logging on every sync event — the same observability discipline I apply in production, now on the TPP side.

Honest scope: a working prototype on banks' sandbox APIs, built as a learning vehicle and AI-PM portfolio piece — and the source of a genuinely dual bank-side + TPP-side view of Open Banking that's rare in the market.
